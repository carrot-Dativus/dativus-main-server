### 프로그램 모듈화와 모듈의 독립성에 대한 설명이다.  (1)모듈의 독립성을 판단하는 두 가지 지표이고 모듈과 모듈간의 상호 의존 정도. (2)모듈 내부의 기능적인 집중 정도이다.모듈의 독립성을 높이기 위해서는 모듈 간의 상호 의존 정도를 나타내는 (1)를 낮추고 모듈이 독립적으로 자체 기능만을 수행하도록 (2)를 높여야 한다. (중요)

답 : 1 : 결합도(Coupling) 2 : 응집도(Cohesion)

데스제외공내(데스  제외 공내) - 왼쪽부터 제일 약한 결합도 상태

자료 결합도(data coupling) : 모듈 간의 인터페이스로 값이 전달되는 경우 (a(10))

스탬프 결합도 (stamp) : 모듈 간의 인터페이스로 배열이나 오브젝트, 스트럭처 등이 전달되는 경우(a(*b))

제어 결합도(control) : 어떻게 처리를 해야한다는 제어 요소가 전달 되는 경우(if(x>10))

외부 결합도(external) : 외부에 있는 다른 모듈에서 참조하는 경우

공통 결합도(common) : 파라미터가 아닌 모듈 밖에 전역 변수를 참조하고 전역 변수를 갱신하는 식으로

상호 작용하는 경우

내용 결합도(content) : 다른 모듈 내부에 잇는 변수나 기능을 다른 모듈에서 사용하는 경우 (스파케티 코드, 외계인 코드, 레거시 코드)

기순통절시논우(우리 논던 시절에 통통한 순대 기가막히게 맛있었다) - 왼쪽부터 응집도가 제일 높음

기능적 응집도(functional) : 모듈 내부의 모든 기능이 단일 목적을 위해 수행(+,-)

순차적 응집도(Sequential) :  모듈 내에서 한 활동으로 나온 출력을 다른 활동이 사용시(출력값 → 입력값)

통신적 응집도(communication) : 동일한 입력과 출력을 사용해 다른 기능 수행하는 활동 모여 있을 경우

절차적 응집도(Procedural) : 모듈 안의 구성 요소들이 그 기능을 순차적으로 수행하는 경우

시간적 응집도(temporal) : 특정 시간에 처리되어야 하는 활동들을 한 모듈에서 처리할 경우

논리적 응집도(logical) : 유사한 성격 갖는 요소들을 한 모듈에서 처리되는 경우

우연적 응집도(coincidental) : 모듈 내부의 각 구성 요소들이 연관성이 없는 경우

### 성적 테이블을 대상으로 아래 요구사항을 적용하여 결과를 출력하는 SQL문을 작성하시오.
요구사항
1. 성적 테이블에서 과목별 평균 점수가 90점 이상인 과목이름, 최소점수, 최대점수를 출력하시오.
2. WHERE 구문은 하지 않는다.
3. GROUP BY, HAVING을 이용한다.
4. 최소점수, 최대 점수 별칭을 위해 AS문을 이용한다.
5. 명령문의 종결 문자인 세미콜론(;)은 생략 가능합니다.

답 : SELECT 과목이름, MIN(점수) AS 최소점수, MAX(점수) AS 최대점수

       FROM 성적

       GROUP BY 과목이름

       HAVING AVG(점수) ≥ 90
